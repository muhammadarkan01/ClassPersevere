Karakteristik Reward dan Action :
- Reward : Reward dilihat sebagai umpan balik terhadap Agent, karena Reward mengalami penundaan maka umpan balik menjadi tertunda. 
- Action : semua tindakan yang mungkin telah diketahui.
RL secara formal : 
- Reward (Rt)
- State (St)
- Action (At)
Trial & Error Systems 1, 2, 3:
- Ainslie & HerrnStein 1974
Environment (Lingkungan) : 
- Aturan Reward dalam lingkungan kadang dideskripsikan dengan jelas
- Ada env yang perlu dipelajari karena tidak ada aturan yang jelas (Trial dan Error / Eksplorasi)
- Improvisasi masih terjadi meski sudah ada aturan
Karakteristik RL :
- Tidak ada kebenaran pada proses pembelajaran / training yang ada hanya hadiah atau hukuman
- Data yang diperoleh (state dan reward) tergantung Action yang di berikan agent
- Variabel waktu dan urutan state yang dipilih agent melalui Action memiliki pengaruh yang penting
Tipe-tipe Environment RL :
- Deterministic Environment vs Stochastic Environment
Deterministik : hasilnya dapat diketahui berdasarkan state saat ini
Stochastic : hasilnya tidak dapat diketahui berdasarkan state saat ini
- Fully observable vs Partially Observable Environment 
Fully observable : ketika agent dapat menentukan state dari sistem setiap saat, dengan state dari sistem dapat diamati seluruhnya
Partially observable : ketika agent tidak dapat menentukan state sistem setiap saat. 
- Discrete Environment vs Continuous Environment
Discrete : ketika state untuk berpindah dari state ke state lainnya terbatas
Continuous : ketika state untuk berpindah dari state ke state lainnya tidak terbatas
- Episodic and non-episodic Environment 
Episodic ( lingkungan non-sekuensial) : tindakan agent saat ini tidak akan mempengaruhi tindakan di masa depan
Non-Episodic (sequential) : tindakan agent saat ini mempengaruhi tindakan di masa depan
- single dan multi-agent Environment 
Single : hanya memiliki agent tunggal 
Multi-agent : memiliki lebih dari 1 agent atau lebih

3 Pendekatan untuk proses seleksi fitur :
1. Metode Fitur
   Metode filter memberi skor untuk setiap fitur , lalu fitur dengan skor tinggi dipilih dan diberi peringkat. Beberapa metode filter yang diusulkan oleh peneliti : Chi-square, Information Gain, Query Expensive Ranking, etc. 
2. Metode Wrapper
   Metode wrapper adalah metode memilih fitur penting menggunakan algoritma optimasi denga algoritma ML. Beberapa metode wrapper yang diusulkan peneliti : Genetic Algorithm, PSO, Iterated Greedy Metaheurestic, etc.
3. Metode Hybrid
   Metode hybrid adalah kombinasi dari metode filter dan wrapper. Metode yang diusulkan peneliti : IG-based PSO, Relief with MVO.

- Image Negatives (linear)
- Log Transformation : mengganti semua nilai piksel dalam gambar dengan nilai algoritmanya dan digunakan untuk peningkatan gambar
- Power law (Gamma) transformation : untuk menampilkan citra secara akurat di komputer dan mengontrol kecerahan gambar
- Contrast Streching

4.) Citra terang : citra yang memiliki banyak piksel dengan nilai intensitas mendekati 255. 
5.) Hidtogram Equalization : Proses meratakan histogram agar derajat keabuan merata antar 0-255 dengan keluaran citra tidak dominan gelap/terang 
Step-step pada histogram equalization : 
1. Hitung distribusi kumulatif dari citra
2. Hitung hasil normalisasi dari histogram
3. Terapkan histogram normalisasi ke citra

   Markov Decision Process merupakan proses Tuple (state, action, transition probability, reward function, dan discount factor). MDP mendeskripsikan secara formal lingkungan RL. 
RL problem dapat diformalisasikan menggunakan MDP : 
- Optimal Control 
- Partially Observable problems
- Bandits problem
Aplikasi RL dalam dunia nyata : 
- Robotik : memutuskan bergerak kemana
- Alokasi sumber daya : memutuskan apa yang diproduksi
- Pertanian : memutuskan apa yang ditanam
Contoh : MDP untuk permainan dadu bisa memilih keluar atau main
- States: himpunan dari states
- 𝑆𝑠𝑡𝑎𝑟𝑡 ∈ 𝑆𝑡𝑎𝑡𝑒𝑠 : state awal
- Actions(s): actions yang tersedia dari state s
- 𝑃 (𝑠, 𝑎, 𝑠′) : probabilitas ke s’ jika mengambil action a dari state s
- Reward (s, a, s’) : idem dengan diatas
- isEnd(s) : apakah state terakhir
- 0 ≤ 𝛾 ≤ 1 faktor diskon
Probabilitas atau Transistion
   Transition Probabilitas 𝑃 (𝑠, 𝑎, 𝑠′) menentukan kemungkinan dari state yang didatangi dalam s’ jika melakukan action a dari state s. Probabilitas memiliki jumlah 1. 
   Policy adalah fungsi 𝜋 yang memetakan untuk setiap state 𝑠 ∈ 𝑆𝑡𝑎𝑡𝑒𝑠 ke action 𝑎 ∈ 𝐴𝑐𝑡𝑖𝑜𝑛𝑠(𝑠). 
   Return (utility) dari sebuah policy adalah Jumlah dari reward selama mengikuti jalur (nilai yang acak). Return adalah 𝐺𝑡 total reward yang di-diskon dari setiap waktu t.
Diskon y : return. 
Evaluasi sebuah policy : value dari policy
Value adalah harapan dari return (Expected return). 
Value dari policy adalah harapan dari return (Expected return) ketika mengikuti suatu policy. 
Kesimpulan algoritma : 
- Evaluasi policy adalah (MDP , 𝜋) -> V 𝜋
- Value iteration adalah MDP -> Vopt, 𝜋opt. Jenis value iteration : dynamic programming.

Karakteristik RL :
- Tidak ada kebenaran pada proses pembelajaran / training yang ada hanya hadiah atau hukuman
- Data yang diperoleh (state dan reward) tergantung Action yang di berikan agent
- Variabel waktu dan urutan state yang dipilih agent melalui Action memiliki pengaruh yang penting
Tipe-tipe Environment RL :
- Deterministic Environment vs Stochastic Environment
Deterministik : hasilnya dapat diketahui berdasarkan state saat ini
Stochastic : hasilnya tidak dapat diketahui berdasarkan state saat ini
- Fully observable vs Partially Observable Environment 
Fully observable : ketika agent dapat menentukan state dari sistem setiap saat, dengan state dari sistem dapat diamati seluruhnya
Partially observable : ketika agent tidak dapat menentukan state sistem setiap saat. 
- Discrete Environment vs Continuous Environment
Discrete : ketika state untuk berpindah dari state ke state lainnya terbatas
Continuous : ketika state untuk berpindah dari state ke state lainnya tidak terbatas
- Episodic and non-episodic Environment 
Episodic ( lingkungan non-sekuensial) : tindakan agent saat ini tidak akan mempengaruhi tindakan di masa depan
Non-Episodic (sequential) : tindakan agent saat ini mempengaruhi tindakan di masa depan
- single dan multi-agent Environment 
Single : hanya memiliki agent tunggal 
Multi-agent : memiliki lebih dari 1 agent atau lebih

Probabilitas atau Transistion
   Transition Probabilitas 𝑃 (𝑠, 𝑎, 𝑠′) menentukan kemungkinan dari state yang didatangi dalam s’ jika melakukan action a dari state s. Probabilitas memiliki jumlah 1. 
   Policy adalah fungsi 𝜋 yang memetakan untuk setiap state 𝑠 ∈ 𝑆𝑡𝑎𝑡𝑒𝑠 ke action 𝑎 ∈ 𝐴𝑐𝑡𝑖𝑜𝑛𝑠(𝑠). 
   Return (utility) dari sebuah policy adalah Jumlah dari reward selama mengikuti jalur (nilai yang acak). Return adalah 𝐺𝑡 total reward yang di-diskon dari setiap waktu t.
Diskon y : return. 
Evaluasi sebuah policy : value dari policy
Value adalah harapan dari return (Expected return). 
Value dari policy adalah harapan dari return (Expected return) ketika mengikuti suatu policy.

Terdapat 2 kelompok penggunaan perintah SQL : 
- Data Definition Language (DDL) : perintah SQL untuk membuat dan memodifikasi objek database. Contoh : create, alter, drop, dan truncate. 
- Data Manipulation Language (DML) : perintah SQL untuk menambah, menghapus, dan memodifikasi data dalam database. Contoh : select, insert, update, dan delete.
